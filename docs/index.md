# CE FranchiseOwner Help

CE FranchiseOwner is the Windows desktop application built for
Commission Express of South Carolina. It runs three products from
one shared database: a lead-generation engine for outbound agent
outreach, a cold-call tracker for manual follow-up, and a
pre-underwriting scorer that reads inbound application emails and
runs them through a 12-question gate check.

## Three Products

### 1. Lead Engine

Scrapes PropStream for Active Under Contract listings across South
Carolina counties, fuzzy-matches the listing agents against a master
roster of licensed SC real-estate agents, then sends a personalised
drip-email sequence to each matched agent. Every outgoing email goes
through Rackspace SMTP and is logged to a single `email_log` source
of truth.

### 2. Call Tracker

Manages the cold-call follow-up workflow for agents who were
previously contacted but not converted. Imports a "Previous Clients"
Excel file, counts deals per agent, fuzzy-matches them against the
master list, lets the franchise owner approve or reject each match,
and then drives a daily call worksheet with outcome tracking
(Interested, Call me back, Left voicemail, Not interested, Do not
call, Converted) and a follow-up date per agent.

### 3. Pre-Underwriting

Receives Commission Express application emails into a dedicated
IMAP mailbox, parses the HTML body into encrypted database rows,
classifies and scans each attached PDF (purchase contract, MLS
sheet, pre-qual letter, etc.), and runs the 12-question gate check
to return a fundability verdict. Two of the questions (Q4 open
contingencies, Q8 pre-approval / proof of funds) use Claude Haiku
4.5 against the attached PDFs; the rest are local scoring.

## How this site is maintained

This site is regenerated by the Docs Engine tab inside the app's
Admin window. The operator clicks **Copy Prompt**, pastes the
prompt into Claude Code in VS Code with the source repository open,
and Claude reads the current state of the code and overwrites the
markdown files in the
[CE-FranchiseOwner-Docs](https://github.com/adssconsulting/CE-FranchiseOwner-Docs)
repository. A GitHub Actions workflow auto-publishes the rebuilt
site to GitHub Pages within two minutes of every push to `main`.

Each successful regeneration appends a line to `logs/doc_run.log`
in the source repo; the Docs Engine tab reads that file to set its
status banner colour (green when up to date, amber while a run is
in progress, red on failure, grey when never run).
